# 本宮市 消防水利マップ（消火栓・防火水槽 点検／訓練管理）

単一のHTMLファイルで動く、本宮市向けの消防水利マップです。消火栓・防火水槽の位置と点検状況、水利空白（火災リスク）の可視化、訓練シミュレーション（水元・火点・中継・展張本数・メモ）、訓練計画の日付保存などができます。

このリポジトリは **GitHub Pages** で公開して、分団内でURLを共有・閲覧するための一式です。

---

## 収録ファイル

- `index.html` … アプリ本体（これ1つで動きます）
- `data.example.json` … 共有データのテンプレート（任意）
- `README.md` … この手順書

---

## 公開手順（GitHub Pages）

所要 5〜10分。GitHubアカウントが必要です（無料）。

1. **リポジトリを作成**
   GitHubにログイン → 右上「＋」→「New repository」。
   - Repository name：例 `motomiya-suiri-map`
   - 公開範囲：後述の「公開範囲の注意」を読んでから **Public / Private** を選択
   - 「Create repository」

2. **ファイルをアップロード**
   作成したリポジトリの「Add file」→「Upload files」→ `index.html` をドラッグ＆ドロップ →「Commit changes」。
   （共有データを使う場合は後述の手順で `data.json` も同じ場所に置きます）

3. **Pages を有効化**
   リポジトリの「Settings」→ 左メニュー「Pages」→
   - Source：**Deploy from a branch**
   - Branch：**main** ／ フォルダ **/(root)** →「Save」

4. **URLを確認**
   数分後、Pagesの画面に公開URLが表示されます。
   形式：`https://<ユーザー名>.github.io/<リポジトリ名>/`
   このURLをスマホで開けば分団の各自が閲覧できます（QRコードにして配ると便利です）。

> HTTPSで公開されるため、スマホの「📍現在地（GPS）」もそのまま使えます。

## 共有データ（全員に同じ地図を見せる）

このアプリのデータは、**各自の端末（ブラウザ）内**に保存されます。つまり、URLを開いただけでは各自が別々の（最初は同じサンプルの）状態から始まり、**誰かの入力が自動で他の人に共有されることはありません。**

分団全員に「同じ消火栓・防火水槽の一覧」を見せたい場合は、次のようにします。

1. 代表者が現地調査してアプリに入力
2. アプリの「⋯データ」→「1) JSON書き出し」でファイルを保存
3. そのファイル名を **`data.json`** に変更
4. `index.html` と同じ場所（リポジトリ直下）にアップロード（Commit）

以後、URLを開いた全員に `data.json` の内容が初期表示されます。内容を更新したいときは、新しい `data.json` を同じ場所に上書きアップロードしてください。

- 端末内にまだ何も保存していない人 … 開くたびに最新の `data.json` が表示されます
- 端末で追加・編集した人 … その端末では「自分の編集」が優先されます（`data.json`更新を取り込みたいときは「⋯データ」→「5) 共有データ(data.json)を再読込」）
- テンプレートは `data.example.json` を参照してください（中身を書き出したJSONに置き換え、名前を `data.json` に）

## 公開範囲の注意（重要）

- **Public リポジトリ＋GitHub Pages は、URLを知っていれば誰でも閲覧できます。** 消火栓・防火水槽の位置や破損状況、写真などをそのまま世界に公開してよいか、分団・本部で必ず確認してください。
- 閲覧を分団内に限りたい場合の選択肢：
  - `data.json`（実データ）は公開せず、`index.html` だけ公開し、各自がJSONを個別に取り込む運用にする
  - リポジトリを **Private** にする（Private リポジトリでの Pages 公開は有料プランが必要な場合があります）
  - 社内・団内の共有ストレージ等に置く
- 個人が写る写真や、住居が特定される情報の扱いにも配慮してください。

## よくある質問

- **地図が出ない／真っ白** … 初回はインターネット接続が必要です（地図タイルと地図ライブラリを読み込みます）。電波のある場所で開いてください。
- **データが消えた** … ブラウザのデータ消去や別ブラウザ利用で端末内保存が失われることがあります。大事な記録は「JSON書き出し」で定期バックアップを。
- **みんなの入力をまとめたい** … 各自の「JSON書き出し」を代表者が集約→統合し、`data.json` を更新する運用が簡単です。将来的に本格的な共同編集が必要なら、スプレッドシートやGIS等への発展も検討できます。
